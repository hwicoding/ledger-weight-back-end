# 서브에이전트 제도 — ledger-weight-back-end

사용자는 **코드를 직접 작성하지 않고**, 에이전트를 적극 활용한 **바이브 코딩**을 한다.  
**모든 권한은 에이전트에게** 있으며, **단독 에이전트가 전부 처리하는 것은 비효율적**이므로, **다중 에이전트 협업**을 기본으로 한다.  
협업 구조는 **SOLID에 맞춘 역할 분리**와 **명확한 입력/출력 계약**으로 최대 효율을 낸다.

---

## 1. 사용자 vs 에이전트

| 구분 | 역할 |
|------|------|
| **사용자** | 의도·방향·요구사항 전달. 코드 작성 없음. 바이브 코딩. |
| **에이전트** | 코드 수정·셸 실행·검색·위임·배포 결정 등 **전권** 보유. 사용자 대신 실행. |

---

## 2. 왜 다중 에이전트인가

- **단독 에이전트**: 탐색 + 수정 + 셸 + 판단을 한 에이전트가 하면 컨텍스트 비대, 속도·정확성 저하.
- **다중 에이전트**: 역할별로 나누고, 메인은 오케스트레이션·통합·최종 결정만. 구체 작업은 서브에이전트에 위임 → **속도·정확성·효율** 극대화.

---

## 3. SOLID를 에이전트에 적용

| 원칙 | 의미 |
|------|------|
| **S (단일 책임)** | explore=탐색만, shell=셸만, generalPurpose=복합 조사/초안만. 다른 타입의 일을 섞지 않음. |
| **O (개방-폐쇄)** | 새 작업은 위임 조합(병렬·연쇄)으로 확장. 에이전트 타입 정의는 고정. |
| **L (리스코프)** | 같은 타입은 동일 계약. 메인은 "explore면 경로/요약, shell이면 명령 결과"만 기대. |
| **I (인터페이스 분리)** | 각 에이전트에 **그 역할에 필요한 정보만** 전달. 과한 맥락 금지. |
| **D (의존성 역전)** | 메인은 "무엇을 할지(태스크)"에 의존. 서브에이전트 내부 구현에 의존하지 않음. |

---

## 4. 에이전트 타입과 계약

| 타입 | 책임 | 입력 | 출력 | 하지 않는 것 |
|------|------|------|------|--------------|
| **explore** | 코드베이스 탐색 | 목적·키워드·디렉터리, thoroughness | 파일 경로·요약·발견 | 수정·셸 |
| **shell** | 셸 명령 실행 | 명령 시퀀스/스크립트, cwd | 결과·exit code | 검색·수정 |
| **generalPurpose** | 복합 조사·초안 | 다단계 작업 설명 | 조사 결과·수정 제안·초안 | 최종 적용(메인이 함) |

---

## 5. 누가 무엇을 할지 (세분화)

### 5.1 메인 에이전트가 직접 할 때 (도구만 사용)

- **Grep/Read**: 경로·심볼을 **이미 알고 있음**, 또는 1~2 키워드·1~3개 파일로 끝.
- **Shell**: 명령 **1~2개**, 연속 의존 없음.

### 5.2 서브에이전트에 위임할 때

- **explore**: "어디에 OO가 있는지 모름", 여러 디렉터리·의미 기반 검색 필요.
- **shell**: 명령 **3개 이상** 또는 git/docker/배포처럼 연속·분기 필요.
- **generalPurpose**: "검색 → 분석 → 초안"이 한 덩어리로 필요.

### 5.3 최종 적용은 항상 메인

- 서브에이전트는 **결과·제안·초안**만 반환. **파일 수정·배포 실행 등 최종 적용은 메인 에이전트**가 수행.

---

## 6. 협업 패턴: 병렬 vs 연쇄

| 패턴 | 언제 | 예시 |
|------|------|------|
| **병렬** | 서로 의존 없음 | explore 2개 동시(예: API 구조 + WebSocket 구조), explore + shell 동시 |
| **연쇄** | B가 A 결과 필요 | explore 완료 → 결과 경로로 메인이 Read 후 수정, 또는 generalPurpose에 "이 경로 기준으로 초안" 전달 |

---

## 7. 규칙 파일과의 관계

- **.cursor/rules/00-project-core.mdc**: 바이브 코딩, 에이전트 전권, 다중 에이전트 기본.
- **.cursor/rules/03-subagents.mdc**: SOLID·계약·도구 vs 위임·병렬/연쇄·메인 책임.
- **AGENTS.md**(이 파일): 제도 개요, SOLID, 계약 표, 세분화 기준, 협업 패턴.

메인 에이전트는 위 규칙과 이 문서를 따라, **단독 처리보다 협업을 우선**하고, **역할·계약을 지키며** 위임한다.
