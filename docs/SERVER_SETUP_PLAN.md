# 서버 구축 작업 계획

**작성일**: 2025-12-11  
**상태**: 준비 중

---

## 📋 개요

프론트엔드와의 연동 테스트를 위한 서버 구축 작업 계획입니다.

---

## 🎯 작업 목표

1. 프론트엔드 요청 형식에 맞는 WebSocket 엔드포인트 구현
2. 로비 연결 및 게임 시작 기능 구현
3. 개발 환경 서버 실행 및 테스트

---

## 📝 작업 항목

### 1. 로비 엔드포인트 추가 (우선순위: 높음)

#### 1.1 `/lobby/{game_id}` WebSocket 엔드포인트 구현
- [ ] 쿼리 파라미터로 `player` 이름 받기
- [ ] 플레이어 ID 자동 생성 (UUID 또는 서버 발급)
- [ ] 로비 상태 관리 (게임 시작 전 대기 상태)
- [ ] 플레이어 참가 시 자동 게임 상태 전송
- [ ] 여러 플레이어 동시 참가 지원

**예상 소요 시간**: 30분

#### 1.2 게임 시작 프로토콜
- [ ] `START_GAME` 메시지 타입 처리
- [ ] 최소 플레이어 수 확인
- [ ] 게임 시작 시 모든 플레이어에게 상태 전송

**예상 소요 시간**: 20분

---

### 2. 서버 실행 환경 설정

#### 2.1 개발 서버 실행
- [ ] 기본 포트 확인 (현재: 8000, 프론트엔드 요청: 8080)
- [ ] 환경 변수 설정 (.env 파일)
- [ ] CORS 설정 확인

**예상 소요 시간**: 10분

#### 2.2 테스트 준비
- [ ] WebSocket 연결 테스트 도구 준비
- [ ] 기본 메시지 송수신 테스트
- [ ] 에러 처리 테스트

**예상 소요 시간**: 15분

---

### 3. 문서화

#### 3.1 API 문서 업데이트
- [ ] WebSocket 엔드포인트 문서화
- [ ] 메시지 프로토콜 상세 문서 작성

## 🔧 기술 스택

- **서버**: FastAPI + Uvicorn
- **WebSocket**: FastAPI WebSocket
- **포트**: 8000 (기본) 또는 8080 (프론트엔드 요청)
- **환경**: Python 3.8+
