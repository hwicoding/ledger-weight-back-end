1. 프로젝트 개요

장부의 무게는 BANG! 카드 게임을 웹으로 구현한 실시간 멀티플레이 백엔드입니다. FastAPI로 HTTP API와 WebSocket을 제공하며, 클라이언트는 로비 접속 → 게임 참가 → 카드 액션을 메시지로 주고받습니다.

파일 역할 요약

  • main.py — FastAPI 앱 생성, CORS, GET /, GET /health, WebSocket /lobby/{game_id}, /ws/{player_id} 정의. 전역으로 GameManager, ConnectionManager, MessageHandler 사용.
  • config.py — pydantic_settings로 .env에서 HOST, PORT, CORS, 게임/WebSocket 상수 로드.
  • game/ — 게임 생성·플레이어 추가·턴·카드 뽑기/사용·승리 판정.
  • websocket/ — 연결 수락/해제, 플레이어별/방별 메시지 전송, 수신 메시지를 MessageHandler로 라우팅.
  • models/ — Pydantic 모델: Game, Player, Card, Role.
  • utils/constants.py — Role, GameState, TurnState, CardType, Suit, Rank 등 Enum.

3. HTTP 요청 흐름

클라이언트가 GET / 또는 GET /health를 호출할 때의 흐름입니다. Uvicorn이 ASGI로 요청을 받아 FastAPI가 라우트에 맞는 핸들러를 실행하고 JSON을 반환합니다.

sequenceDiagram
  participant C as 클라이언트
  participant U as Uvicorn
  participant F as FastAPI
  participant R as 라우트 핸들러

  C->>U: GET /health
  U->>F: ASGI 요청
  F->>F: CORS 미들웨어
  F->>R: health_check()
  R->>R: return {"status": "healthy"}
  R->>F: JSON 응답
  F->>U: ASGI 응답
  U->>C: 200 OK + JSON

4. WebSocket 메시지 흐름

로비 엔드포인트 /lobby/{game_id}?player=이름 접속 시 연결 수락 → 자동 참가 → 이후 클라이언트 메시지(JOIN_GAME, PLAYER_ACTION, GET_GAME_STATE 등)를 MessageHandler가 처리하고, 필요 시 해당 게임 플레이어들에게 브로드캐스트합니다.

sequenceDiagram
  participant C as 클라이언트
  participant M as main.py
  participant Conn as ConnectionManager
  participant Msg as MessageHandler
  participant GM as GameManager

  C->>M: WS /lobby/game_1?player=홍길동
  M->>Conn: connect(ws, player_id)
  Conn->>C: accept()
  M->>C: CONNECTION_ESTABLISHED
  M->>Msg: handle_join_game(player_id, JOIN_GAME)
  Msg->>GM: add_player_to_game / create_game
  GM-->>Msg: 성공/실패
  Msg-->>M: join_result
  M->>C: ERROR 또는 계속

  loop 메시지 루프
    C->>M: JSON 메시지
    M->>Msg: handle_message(player_id, message)
    alt PLAYER_ACTION
      Msg->>GM: (action_handler 등)
      GM-->>Msg: 결과
    end
    Msg-->>M: result
    M->>Conn: send_personal_message / broadcast
    Conn->>C: JSON 응답
  end

  C->>M: 연결 종료
  M->>Conn: disconnect(player_id)

메시지 타입

message_handler.handle_message에서 type에 따라 분기합니다.

  • JOIN_GAME — 게임 참가 (로비 접속 시 자동 호출)
  • START_GAME — 게임 시작
  • GET_GAME_STATE — 현재 게임 상태 조회
  • PLAYER_ACTION — 카드 사용 등 액션 (action_handler 연동)
  • ADD_AI_PLAYER — AI 플레이어 추가

5. Python 개념과 프로젝트 내 활용

이 프로젝트에서 쓰이는 Python 개념을 짧게 정리하고, 어디서 사용되는지 매핑했습니다.

async / await 비동기 함수. I/O 대기 시 다른 코루틴이 실행되어 동시 접속을 효율적으로 처리합니다.
사용처: main.py 라우트, connection_manager, message_handler
타입 힌트 (Type Hints) 변수·인자·반환 타입을 str, Dict, Optional[T] 등으로 표기. 가독성과 도구 지원에 유리합니다.
사용처: 전 모듈 (game, models, websocket)
Enum 고정된 상수 집합(역할, 게임 상태, 카드 타입 등)을 타입으로 정의합니다.
사용처: utils/constants.py (Role, GameState, TurnState, CardType)
클래스와 인스턴스 GameManager, ConnectionManager, MessageHandler 등이 상태를 갖는 객체로, main에서 한 번 생성해 전역 사용합니다.
사용처: app 전역 인스턴스
딕셔너리 / set games: Dict[str, Game], active_connections: Dict[str, WebSocket], game_players: Dict[str, Set[str]] 등으로 관계를 저장합니다.
사용처: GameManager, ConnectionManager
모듈 / 패키지 app.game, app.models, app.websocket 등으로 역할별로 나누어 import 합니다.
사용처: from app.xxx import yyy

6. FastAPI 개념과 프로젝트 내 활용

FastAPI의 핵심 개념과 이 레포에서의 사용 위치를 대응했습니다.

FastAPI() 앱 인스턴스 모든 라우트·미들웨어·의존성의 진입점. app.main:app으로 uvicorn이 로드합니다.
사용처: main.py
라우트 데코레이터 @app.get("/"), @app.get("/health"), @app.websocket("/lobby/...") 로 경로와 HTTP 메서드·WebSocket을 연결합니다.
사용처: main.py
미들웨어 CORSMiddleware로 브라우저 cross-origin 요청을 허용합니다.
사용처: main.py add_middleware
WebSocket WebSocket 타입으로 연결을 받고, accept(), receive_text(), send_*로 양방향 통신합니다.
사용처: main.py, connection_manager
Query / Path 쿼리 파라미터(player=...), 경로 파라미터(game_id, player_id)를 함수 인자로 받습니다.
사용처: lobby_websocket_endpoint(game_id, player=Query(...))
자동 문서화 docs_url="/docs", redoc_url="/redoc" 로 Swagger UI / ReDoc 자동 생성.
사용처: main.py FastAPI(...)

7. 라이브러리별 개념과 활용

requirements.txt 기준으로, 각 라이브러리가 무엇이고 이 프로젝트에서 어디에 쓰이는지 정리했습니다.

FastAPI

웹 프레임워크

역할: HTTP·WebSocket 라우팅, 요청/응답, 자동 스키마 생성.

활용: main.py에서 앱 생성, GET/WS 엔드포인트, CORS, WebSocket, Query 사용.

Uvicorn

ASGI 서버

역할: ASGI 앱을 실행해 네트워크 I/O를 처리하고, async 라이프사이클을 돌립니다.

활용: uvicorn app.main:app --host 0.0.0.0 --port 8088 로 서버 기동. main.py의 if __name__ == "__main__"에서도 호출.

Pydantic

데이터 검증·설정

역할: 모델 클래스로 타입·검증·직렬화를 정의합니다.

활용: models/의 Game, Player, Card, Role — 필드 타입과 기본값. API 스키마 자동 생성에 사용.

pydantic-settings

설정 로드

역할: 환경 변수·.env 파일을 Pydantic 모델로 로드합니다.

활용: config.py의 Settings(BaseSettings) — HOST, PORT, CORS_ORIGINS, WS_MAX_CONNECTIONS 등.

python-dotenv

.env 로드

역할: 프로젝트 루트의 .env를 읽어 os.environ에 넣습니다.

활용: pydantic-settings가 .env를 쓰도록 할 때 함께 사용. Settings의 env_file=".env"와 연동.

websockets

WebSocket 프로토콜

역할: WebSocket 핸드셰이크·프레임 처리를 위한 저수준 라이브러리.

활용: FastAPI/Starlette이 내부적으로 사용. 이 프로젝트에서는 FastAPI의 WebSocket API만 직접 사용.

python-multipart

폼·파일 업로드

역할: multipart/form-data 파싱.

활용: 파일 업로드나 폼 필드가 있는 엔드포인트를 쓸 때 FastAPI가 사용. 현재 프로젝트에서는 사용하지 않아도 의존성으로 포함 가능.

loguru

로깅

역할: 구조화된 로그, 파일·레벨 설정이 쉬운 로거.

활용: 필요 시 from loguru import logger로 예외·디버그 로그 출력. (현재 코드에서는 print/traceback도 사용)

pytest / pytest-asyncio

테스트

역할: 단위·통합 테스트, async 테스트 실행.

활용: tests/에서 FastAPI TestClient, 게임 로직 테스트 등에 사용.

python-jose / passlib

JWT·비밀번호 해싱

역할: JWT 발급·검증, bcrypt 등 비밀번호 해싱.

활용: 인증·로그인 기능을 추가할 때 사용. 현재 게임 API에는 미사용.
